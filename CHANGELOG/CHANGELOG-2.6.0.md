# v2.6.0

## Changes by Kind

### Feature

- New daemonset that can run in Windows nodes ([#275](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/275), [@mauriciopoppe](https://github.com/mauriciopoppe))
- Support for multi Windows distro builds ([#273](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/273), [@mauriciopoppe](https://github.com/mauriciopoppe))
- Support for watching ConfigMap changes and restarting main controller ([#265](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/265), [@yibozhuang](https://github.com/yibozhuang))

### Bug or Regression

- Users are prevented from creating new physical volumes for local static paths that are used in existing volumes.  An attempt to do so will return an error indicating the path and the existing volume which are preventing the new volume from being created ([#246](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/246), [@davidmccormick](https://github.com/davidmccormick))

### Other (Cleanup or Flake)

- Bump base image to k8s.gcr.io/build-image/debian-base-amd64:buster-v1.10.0 ([#295](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/295), [@mauriciopoppe](https://github.com/mauriciopoppe))
- ClusterRole system:persistent-volume-provisioner replaced with a custom ClusterRole with the same contents minus permissions to access PVCs ([#292](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/292), [@mauriciopoppe](https://github.com/mauriciopoppe))
- Bump kubernetes version from 1.22.1 to 1.24.0-alpha.4 ([#294](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/294), [@mauriciopoppe](https://github.com/mauriciopoppe))
- Upgraded Kubernetes dependencies to 1.22.1 ([#264](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/264), [@dobsonj](https://github.com/dobsonj))

### Uncategorized

- Release chart as a package and publish it on gh-pages branch, you can now pull the provisioner helm chart via our repository: https://kubernetes-sigs.github.io/helm/sig-storage-local-static-provisioner ([#280](https://github.com/kubernetes-sigs/sig-storage-local-static-provisioner/pull/280), [@skylenet](https://github.com/skylenet))
